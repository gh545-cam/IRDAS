## 🎯 Key Capabilities

### Real Vehicle Simulation
- Creates synthetic "real" vehicle data with parameter mismatch
- Tire coefficient variations ±10%
- Mass variations ±5%
- Aerodynamic coefficient variations ±5%
- Reproducible with seeds

### Kalman Filtering
- Extended Kalman Filter for nonlinear dynamics
- Sensor noise modeling (IMU, GPS, wheel speeds)
- Process and measurement noise covariance tuning
- Uncertainty propagation and state confidence bounds
- Realistic IMU bias and GPS latency handling

### Neural Network Learning
- Residual dynamics architecture (learns correction, not full model)
- PyTorch implementation with modern best practices
- Batch normalization for training stability
- Dropout for regularization
- Early stopping to prevent overfitting
- Adam optimizer with gradient clipping

### Parameter Adaptation
- Recursive Least Squares for online efficiency
- 7 key parameters: tire stiffness/peak friction, mass, aerodynamics
- Physically-constrained optimization
- Forgetting factor for time-weighted learning
- Convergence analysis and tracking

## 🎓 Understanding the System

### System Flow
```
1. Read Control Input
   ↓
2. Real Vehicle Simulator
   ├─ Generates ground truth (real) state
   ├─ Uses mismatched parameters
   └─ Simulates what real car would do
   ↓
3. Sensor Simulation
   ├─ Add measurement noise
   └─ Create realistic measurements
   ↓
4. Kalman Filter
   ├─ Predict: Use baseline model
   ├─ Update: Fuse with measurements
   └─ Output: State estimate + uncertainty
   ↓
5. Neural Network (Optional)
   ├─ Predict residual correction
   └─ Improve model predictions
   ↓
6. Parameter Adapter
   ├─ Observe model error
   ├─ Update parameters (RLS)
   └─ Improve baseline model
   ↓
7. Output
   ├─ Estimated state
   ├─ Adapted parameters
   ├─ Model error metrics
   └─ Performance tracking
```

## 🎯 System Philosophy

IRDAS is built on the principle of **modular online learning**:

- **Modular**: Each component (KF, NN, RLS) can be used independently
- **Online**: Adapts in real-time without batch retraining
- **Learning**: Continuously improves from streaming data
- **Practical**: Designed for racing/dynamic driving scenarios
- **Interpretable**: Clear model corrections vs learned residuals
- **Efficient**: Optimized for real-time performance
